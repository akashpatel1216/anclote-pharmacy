## üé® Image Specifications & Best Practices

### Format Requirements
- **Hero backgrounds:** JPG (optimized for web, 100-200KB)
- **Content images:** JPG or PNG (optimized for web, 50-150KB)
- **Logo:** PNG with transparent background

### Optimization Tips
1. Use image compression tools:
   - **TinyPNG** (tinypng.com) - excellent for PNG/JPG compression
   - **Squoosh** (squoosh.app) - Google's image optimization tool
   - **ImageOptim** - Mac app for image optimization

2. Recommended settings:
   - **Quality:** 80-85% (good balance of quality and size)
   - **Format:** JPG for photos, PNG for graphics/logos
   - **Resolution:** 72 DPI for web

### Image Style Guidelines
- **Hero backgrounds:** Slightly muted/desaturated works best (will have overlay)
- **Content images:** Bright, clear, professional-looking
- **Color scheme:** Blues, greens, whites work well with pharmacy theme
- **Avoid:** Overly busy images, stock photos that look too "stocky"

## üí° Pro Tips

1. **Use Real Photos:** If possible, take photos of your actual pharmacy. This adds authenticity!

2. **Consistent Style:** Try to keep a consistent photography style (lighting, color tone) across all images.

3. **Mobile First:** Remember images will be viewed on phones - ensure key elements are visible even when cropped.

4. **Test Locally:** After adding images, test your website locally with `npm run dev` to see how they look.

5. **Fallback:** If an image is missing, the page will still work - the image just won't display. Prioritize essential images first.
